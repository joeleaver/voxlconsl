# voxdude

A voxel pacman clone. Eat dots, dodge ghosts, grab power pellets to
chase them back. Demonstrates a full game loop: multi-actor
coordination, grid-locked movement, ghost AI with four
classic-pacman personalities, MIDI music with cart-side chiptune
synth patches, particles, camera-relative HUD, and a win/lose flow.

## What it does

Top-down voxel maze (28×31 cells). The player roams the corridors
eating dots and dodging four ghosts; one of the four power pellets
flips the ghosts into a frightened state so the player can eat
*them* instead.

Clear every dot to win → maze flashes green and freezes.
Lose three lives → maze flashes red and freezes. **K** restarts.

Music: a bundled 109-second SMF song with 17 tracks (drums + 14
melodic channels) routed through five custom chiptune patches
(triangle bass, growly saw bass, square lead, saw lead, slow-attack
pad). Music starts as soon as the player clicks the canvas to engage
pointer lock.

## Controls

| Key | Action |
|---|---|
| WASD | Move (cardinal-direction grid-locked) |
| K | Restart after win/lose |
| Click canvas | Engage pointer lock + start audio |
| Esc | Release pointer lock |

## What it teaches you

| Topic | Where to look |
|---|---|
| Grid-locked movement with desired-direction buffering | `player.rs::tick_movement` |
| Classic pacman ghost AI with personality targets | `ghosts.rs::pick_dir` + `target_cell` |
| Frightened mode + white-flash tell | `ghosts.rs::trigger_frightened` + `tick_visuals` |
| Per-frame actor repaint with caching | `ghosts.rs::tick_visuals` + `repaint_ghost` |
| Pooled particle system (pre-spawn + reuse) | `particles.rs` |
| Camera-relative HUD (score + lives) | `hud.rs::tick` |
| Score rasterised straight into actor voxels via FONT_ANSI | `hud.rs::paint_digit` |
| MIDI music with cart-side `program_change` chiptune routing | `audio.rs::init_music` + `audio/patches.toml` |
| Material recolour for win/lose flash | `lib.rs::recolor_walls` |

## Building

```sh
./scripts/build-web.sh release voxdude
```

Then open `web/index.html` via `python3 scripts/dev-server.py 8765`
and pick `voxdude` from the cart library.

## Audio pipeline

The music source is a multi-track SMF generated by an external
stems-to-MIDI tool. To prepare it for voxdude:

1. Drop the source MIDI at `~/dev/stems2midi/output/Voxel Synapse Stems.mid`
   (or pass a path to the script).
2. Run `python3 examples/voxdude/audio/preprocess_song.py`.
   It strips every `program_change` event so cart-side
   `audio::program_change` calls in `init_music()` win.
3. The bundler picks up the cleaned MIDI from `audio/songs/` and packs
   it into the cart's Audio section.

Patch slots:

| Slot | Patch | Used for |
|---|---|---|
| 0 | chomp | Dot pickup SFX |
| 1 | ping | Power-pellet / ghost-eaten / win / death SFX |
| 2 | tri_bass | Bass: Piano / Bass / Strings / Synth Pad channels |
| 3 | saw_bass | Bass: Organ / Guitar / Synth Lead channels |
| 4 | square_lead | Synth: Synth Lead / Brass / Reed channels |
| 5 | saw_lead | Synth: Piano / Guitar (clean) channels |
| 6 | pad | Synth: Organ / Strings channels (sustained polyphonic wash) |
